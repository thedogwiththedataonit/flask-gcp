What is

GCP Monitor?

GCP Native cloud monitoring platform


Offerings

Metrics

Monitoring

Uptime Checks

Health and latency of services, web pages, APIs

Custom Metrics

Application specific metrics (count of returning users

Log-based Metrics

Content count of certain log entries (count of error logs)

Agent Metrics

Installed onto VMs (AWS as well) for system level metrics.

Alerting

GCP

Alert Policies

Tied closely to metrics. A set of conditions that will generate a notification whenever that event is triggered. The notification can contain documentation and metadata to provide additional info. Email, PagerDuty, SMS, Hipchat, Webhooks, GCP Mobile APP

Error Reporting

Groups together error and critical level errors from application logs. Sends a summary of what, where, when, and why an incident occurred.

Debugger

GCP

Inspect the state of an application


Snapshots

Snapshots capture local variables and the call stack at a specific line location in your app's source code.

Logpoints

Logpoints allow you to inject logging into running services without restarting or interfering with the normal function of the service.

Traces

GCP

Flame Graph


Latency data in ms on GCP services. Reveals requests over a timespan, clicking onto the individual requests show the underlying Logs
Filter by services or requests to see how latency has chanced over time

Logging

GCP

Query Builder

Menu selector to filter by resource, log type, log entry OR write queries in query language to filter logs.

Query Results

All GCP services are sent to the Cloud Logging API which is then filtered by the Log Router to sort the logs in Log Storage, Cloud Storage, BigQuery. Logs are retained by default for 30 days and max for 3650 days.

Profiler

GCP

Flame Graph

Similar to debugging, the GCP profiler uses a flame graph to continuously analyze the performance of CPU or memory intensive functions executed across an application. Cloud Profiler presents the call hierarchy and resource consumption of the relevant function in an interactive flame graph that helps developers understand which paths consume the most resources and the different ways in which their code is actually called.

Why

Datadog?


Agent Installation

GCP Cloud Monitor requires an installation on the VM similar to Datadog's agent. While GCP can monitor both hybrid and multi-cloud environments, Datadog has superior granularity.

Superior Metrics and Integrations

Datadog has over 50 metrics enabled by default, more metrics can be added with Datadog's application-specific integrations. Datadog offers more supported integrations than GCP monitor.

No Synthetics or RUM

GCP does not offer any synthetics or Real User Monitoring. They offer debugging and profiling for applications and services, however, they do not support any front-end monitoring solutions like Datadog. On top of this, Datadog has superior CI/CD monitoring, security, and social aspects on their platform.

Superior DevOps Communciation

Datadog offer public invite urls which allows anyone on any team to gain value from the platform. Datadogs social feed of alerts, incidents, and graphs can be shared within the platform and even across Pagerduty, Slack, ServiceNow.

Unified Tagging Structure

GCP requires a query builder (menu selector and writing queries) to filter through logs. Datadog uses a unified tagging and facet structure that seamlessly filters through logs and ties them to relevant instances or services.

Superior UI

GCP Monitor's dashboards and UI are nothing special.
